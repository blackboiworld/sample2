Some common Block Tags

hr

The <hr> element used to stand for "horizontal line" and that is still how it is typically displayed by browsers. However, in HTML5 the tag took on a semantic meaning, "semantic break." I will be using it often to break up example files when I want to display multiple tags in one file.

Using the hr tag in HTML5.

Address

The <address> element indicates that the text is an address specific to the page, perhaps contact information for the author

1, Aro street
Ilogbo - Elegba
Ojo, Lagos state.
102110.

Using the address tag in HTML5.

Blockquote

The <blockquote> element is used for extended quotations. it should include the cite attribute in order to show or link to the source of the quote.

"If you cannot do great things, do small things in a great way." Napolean Hills.

Using the blockquote tag in HTML5.

details

The <details> element is used in conjunction with the <summary> tag. It is one of the tags that was created in response to the repeated use of scripting to open and close content. It is not supported by all the browsers yet though.

The summary

The rest of the details content goes here. If only <details> element is not supported, then the summary and the content are always visible.


Using the details tag in HTML5

Inline Elements

Why do we bother with inline elements? There are certainly times we want to mark up content, but we do not want to insert the line breaks around them. The <span> tag was the first to give you that ability, but it was mostly used to style the code, and we do not want to do that. Some of the newer tags that still separate content ans style are listed below. Remember, Inline tags can flow next to each other and do not cause a line break before and after the element. we have seen this with links and images.

Read more on Block and Inline elements here

Citing a source - cite

The <cite> makes it more easier for people and bots to search the code for this specific information.

The Scream by Edward Munch. Painted in 1893.


Using the cite tag in HTML5.

Defining your abbreviations - abbr

The <abbr> will display an abbreviation on the browser. But if a tile attribute is specified in the tag, the title will appear whenthe mouse hovers over the abbreviation. Screen readers also provide the title information.

The days of the week are often abbreviated: Sun,Mon,Tues etc.

Using the abbreviation tag in HTML5.

Specifying the use of time - time

The <time> tag is again a completely semantic tag that can be used to search for specific information.

The party starts at 9:00pm and ends at 12:00am.

The party took place on New Year's Eve.

Using the time tag on HTML5.

Identifying code fragments - code

The <code> element tends to be displayed in a different font, but also has the semantic meaning to convey information to as many people as possible.

I have been trying to learn how to write html but my img element never seem to work.

Using the code tag in HTML5.

Using Subscripts and Superscripts - sub and sup

The <sub> and <sup> elements are used to create the super and subscripts, e.g (x+y)3 or H2O. But images did not convey the same meaning to all users.

Using super and subscripts in HTML5.
